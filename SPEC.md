# KoreaNomad 웹사이트 개선 실행 계획

> **프로젝트 개요**: 한국 디지털 노마드를 위한 도시 추천/커뮤니티 플랫폼
> **스택**: Next.js 16, React 19, TypeScript, Tailwind CSS 4, Shadcn UI, Supabase Auth
> **원칙**: 데이터베이스 없이 목데이터 사용, 각 페이즈는 독립 실행 가능한 완성 상태

---

## 페이즈 로드맵

| 페이즈 | 핵심 목표 | 주요 산출물 |
|--------|-----------|------------|
| **1** | 홈페이지 & 공통 UX 기반 | 검색 동작, 로딩 UI, SEO, 404/에러 페이지 |
| **2** | 도시 탐색 완성 | 필터/정렬 동작, 즐겨찾기, 도시 비교, 탭 상세 |
| **3** | 커뮤니티 인터랙션 | Q&A CRUD, 밋업 참가, 체크인 |
| **4** | 마이페이지 완성 | 프로필 편집, 리뷰 관리, 통계, 온보딩 |
| **5** | 지도 구현 | 인터랙티브 지도, 마커, 히트맵 |
| **6** | 폴리싱 & 최적화 | 성능, 접근성, 애니메이션, 모바일 UX |

## 페이즈 1 — 홈페이지 & 공통 UX 기반 강화

## 페이즈 2 — 도시 탐색 & 상세 페이지 기능 완성

### 오버뷰

도시 관련 페이지는 현재 UI는 갖추고 있으나 필터/검색이 실제로 동작하지 않고, 즐겨찾기는 버튼만 있을 뿐 상태가 저장되지 않습니다. 이 페이즈에서는 사용자가 원하는 도시를 빠르게 찾을 수 있도록 필터/검색/정렬 기능을 완전히 동작시키고, 즐겨찾기를 `localStorage`로 영속화하며, 도시 상세 페이지를 정보 밀도 높은 완성형 페이지로 만듭니다.

**작업 범위**: `app/cities/*`, `components/cities/*`, `lib/mock-data.ts`

### 수정/개선해야 하는 기능

**FilterBar 실제 동작 구현**
- [ ] 페이즈 1에서 연결된 URL 쿼리스트링(`?search=`, `?region=`, `?tag=`, `?sort=`)을 읽어 도시 목록 필터링
- [ ] 필터 조건 복합 적용 (지역 + 특성 + 생활비 범위 + 인터넷 속도 동시 적용)
- [ ] 필터 초기화 버튼 기능 구현
- [ ] 필터 결과 없을 때 "조건에 맞는 도시가 없습니다" 빈 상태 UI 표시
- [ ] 적용된 필터 태그를 상단에 칩(Chip) 형태로 표시, 개별 제거 가능

**도시 목록 정렬 기능 구현**
- [ ] 7가지 정렬 기준 실제 동작 (종합점수/생활비/인터넷/카페/기후/리뷰수/현재체류)
- [ ] 정렬 방향 토글 (오름차순/내림차순) 버튼 추가

**즐겨찾기 기능 (localStorage)**
- [ ] CityCard의 즐겨찾기 하트 버튼 클릭 시 `localStorage`에 도시 ID 저장/제거
- [ ] 즐겨찾기 상태가 페이지 새로고침 후에도 유지되는지 확인
- [ ] 즐겨찾기 추가/제거 시 토스트 알림 표시 (`sonner` 또는 자체 Toast 컴포넌트)
- [ ] `/cities?filter=favorites` 쿼리로 즐겨찾기 도시만 보기 기능

**도시 비교 기능 (목데이터)**
- [ ] 도시 카드에 "비교 추가" 체크박스 추가 (최대 3개 선택 가능)
- [ ] 하단 고정 비교 바 (선택된 도시 썸네일 + "비교하기" 버튼)
- [ ] 비교 모달: 선택된 도시들의 7개 점수 항목을 테이블/레이더 차트로 나란히 표시

**도시 상세 페이지 개선**
- [ ] 탭 구조 개선: "개요 / 생활비 / 리뷰 / 카페&공간" 4개 탭으로 재편
- [ ] "카페&공간" 탭 추가: 목데이터로 추천 카페/코워킹스페이스 카드 3-5개 표시
- [ ] 리뷰 섹션 필터: 직업 유형별 필터 (개발자/디자이너/마케터 등)
- [ ] 리뷰 작성 버튼 클릭 시 비로그인 상태면 로그인 안내 모달 표시
- [ ] "이 도시 공유하기" 버튼 → 클립보드에 URL 복사 + 토스트 알림
- [ ] 인근 추천 도시 3개 (같은 지역 또는 비슷한 점수대) 하단 표시

**목데이터 보강**
- [ ] 각 도시별 추천 카페/코워킹 공간 데이터 3-5개 추가 (`lib/mock-data.ts`)
- [ ] 리뷰 데이터를 도시당 최소 3개로 보강

**필터/검색 검증**
- [ ] 지역 필터 "영남권" 선택 시 부산, 경주, 포항, 통영만 표시되는지 확인
- [ ] 생활비 슬라이더를 "100만원 이하"로 설정 시 해당 도시만 필터링되는지 확인

- [ ] 필터 조건에 맞는 도시가 0개일 때 빈 상태 UI 표시 확인
- [ ] 적용된 필터 칩에서 "X" 클릭 시 해당 필터만 제거되는지 확인

**즐겨찾기 검증**
- [ ] 하트 버튼 클릭 → 빨간 하트로 변경 + 토스트 알림 확인

- [ ] 비교 모달에서 레이더 차트에 3개 도시 데이터가 겹쳐 표시되는지 확인
- [ ] 비교 바 "X" 클릭 시 해당 도시 비교에서 제거되는지 확인

## 페이즈 3 — 커뮤니티 기능 구현 (목데이터 기반)

### 오버뷰

커뮤니티 섹션(`/community`, `/community/qa`, `/community/meetup`)은 현재 UI만 있고 실제 사용자 인터랙션이 전혀 없는 상태입니다. Q&A 게시판은 글 목록만 표시되고 상세 보기, 글 작성, 답변 달기가 모두 미구현입니다. 밋업도 목록만 있고 참가 신청이 없습니다. 이 페이즈에서는 목데이터를 활용해 커뮤니티의 핵심 인터랙션을 모두 구현하고 체크인 기능을 시뮬레이션합니다.

**작업 범위**: `app/community/*`, 신규 컴포넌트 `components/community/*`

### 수정/개선해야 하는 기능

**Q&A 게시판 완성**
- [ ] Q&A 목록 페이지: 카테고리 탭 필터 (전체/숙소/카페/교통/비용/기타) 동작
- [ ] Q&A 목록 페이지: 도시별 필터 드롭다운 동작
- [ ] Q&A 목록 페이지: 검색 기능 (제목/내용 포함 여부)
- [ ] Q&A 상세 페이지 (`/community/qa/[id]`) 신규 생성
  - [ ] 질문 상세 내용 표시
  - [ ] 답변 목록 표시 (목데이터 답변 2-3개)
  - [ ] 답변 작성 폼 (비로그인 시 로그인 안내)
  - [ ] 답변 채택 버튼 (작성자만 표시)
  - [ ] 조회수 카운트 표시
- [ ] Q&A 글 작성 페이지 (`/community/qa/new`) 신규 생성
  - [ ] 제목, 도시 선택, 카테고리, 내용 입력 폼
  - [ ] 비로그인 시 접근 불가 (로그인 페이지로 리다이렉트)
  - [ ] 작성 완료 시 목데이터에 임시 추가 후 목록으로 이동 (`useState` 활용)

**밋업 기능 완성**
- [ ] 밋업 목록 페이지: 도시별 필터, 날짜 범위 필터 동작
- [ ] 밋업 상세 페이지 (`/community/meetup/[id]`) 신규 생성
  - [ ] 밋업 상세 정보 (장소, 시간, 참가자 목록 미리보기)
  - [ ] 참가 신청 버튼 (클릭 시 참가자 수 +1, 최대 정원 도달 시 비활성화)
  - [ ] 참가 취소 버튼 (신청 후 재클릭)
  - [ ] 카카오맵/네이버맵 링크 버튼 (장소명으로 검색 URL 생성)
- [ ] 밋업 생성 페이지 (`/community/meetup/new`) 신규 생성
  - [ ] 제목, 도시, 날짜/시간, 장소, 최대 인원, 설명 입력 폼
  - [ ] 비로그인 시 접근 불가

**체크인 기능 시뮬레이션**
- [ ] 커뮤니티 홈의 "실시간 체크인" 섹션 인터랙션 구현
  - [ ] "체크인" 버튼 클릭 시 도시 선택 모달 표시
  - [ ] 도시 선택 후 해당 도시 `currentNomads` 수 +1 (클라이언트 상태)
  - [ ] 체크인 후 버튼이 "체크아웃"으로 변경
  - [ ] 체크인 상태를 `localStorage`에 저장 (새로고침 유지)

**커뮤니티 홈 개선**

### 작업 완료 후 검증/확인 체크리스트

**Q&A 검증**
